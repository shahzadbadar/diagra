# Diagra Examples

## AWS
- `aws/01-serverless-api.diagra` - Serverless REST API with CloudFront, API Gateway, Lambda, DynamoDB, S3, Cognito, CloudWatch, and X-Ray.
- `aws/02-event-driven.diagra` - EventBridge-driven microservices with SNS, SQS, Lambda consumers, a DLQ, and CloudWatch.
- `aws/03-data-pipeline.diagra` - Analytics pipeline with S3, Kinesis, Glue, Athena, Redshift, SageMaker, and BI dashboards.
- `aws/04-container-platform.diagra` - Container platform with Route53, CloudFront, ELB, VPC, ECS, Fargate, RDS, and Redis cache.
- `aws/05-ml-platform.diagra` - ML training and inference path using S3, SageMaker, Lambda, API Gateway, and CloudWatch.
- `aws/06-security-architecture.diagra` - Security architecture with WAF, CloudFront, API Gateway, Cognito, IAM, KMS, Secrets Manager, and CloudWatch.
- `aws/07-multi-region.diagra` - Active-active multi-region routing with Route53 and DynamoDB global table replication.

## Azure
- `azure/01-web-application.diagra` - Azure web app with Front Door, App Service, Functions, Cosmos DB, Blob Storage, and Application Insights.
- `azure/02-microservices.diagra` - Azure Functions microservices behind API Management with Service Bus, Cosmos DB, Key Vault, and Monitor.
- `azure/03-data-platform.diagra` - Azure data platform with Data Factory, Data Lake, Databricks, Synapse, Power BI, and Monitor.
- `azure/04-aks-platform.diagra` - AKS platform with container registry, ingress, services, sidecars, Cosmos DB, Service Bus, and Monitor.
- `azure/05-ai-platform.diagra` - Azure AI platform with API Management, Functions, Blob Storage, AI Search, Azure OpenAI, Cosmos DB, and Monitor.

## GCP
- `gcp/01-serverless-api.diagra` - GCP serverless API with API Gateway, Cloud Run, Firestore, Cloud Storage, Pub/Sub, and Monitoring.
- `gcp/02-data-pipeline.diagra` - GCP data pipeline with Pub/Sub, Cloud Composer, Cloud Storage, Dataflow, BigQuery, Looker, and Monitoring.
- `gcp/03-ml-platform.diagra` - Vertex AI ML platform with BigQuery ML, Cloud Storage artifacts, Cloud Run, Pub/Sub, and Monitoring.
- `gcp/04-microservices.diagra` - GKE microservices with load balancing, service workloads, Firestore, Pub/Sub, Monitoring, and Trace.

## AI
- `ai/01-claude-code-agent.diagra` - Agentic coding loop for reading code, editing files, running tests, and committing changes.
- `ai/02-rag-pipeline.diagra` - Retrieval augmented generation pipeline with ingestion and online query paths.
- `ai/03-multi-agent.diagra` - Multi-agent orchestration with research, code, review, tools, memory, and final response.
- `ai/04-n8n-workflow.diagra` - n8n AI automation workflow with retries, transforms, notifications, and persistence.
- `ai/05-llm-gateway.diagra` - Multi-model LLM gateway with routing, caching, cost tracking, and logging.
- `ai/06-diagra-architecture.diagra` - Diagra rendering pipeline from `.diagra` source to exported artifacts.

## General
- `general/01-simple-flow.diagra` - Minimal five-node quickstart flow.
- `general/02-microservices.diagra` - Cloud-neutral microservices with per-service databases and async messaging.
- `general/03-cicd-pipeline.diagra` - CI/CD pipeline from git push to production monitoring.
- `general/04-system-design.diagra` - Classic scalable system design with CDN, load balancer, web tier, cache, database, replicas, and object storage.
- `general/05-auth-flow.diagra` - OAuth/JWT authentication flow rendered as a flowchart.
